NLP-SQL Agent
1.用户提问。2.获取提问并更新状态  3.在记忆库里，检索最相似的三个提问转sql问答对增强提示词。4.检索MDL（数据库建模语言）文件增强提示词。5.获取WrenAI框架里面的系统提示词，整合前面的提示词作为llm生成sql语句的上下文，结构化输出sql。6.进行sql复杂度判断，如果不复杂进入执行sql节点执行sql并得到结果，然后根据提问和结果生成自然语言答案，最后存入问答记忆库；如果复杂则进入验证sql语句节点验证（怎么验证的呢？1.检查sql语法，有没有多余的符号。2.判断查询的列、表在MDL里面有没有定义。3.看能不能生成物理层的sql（能不能在数据库中找到真实的表），），如果验证成功就进入执行节点，也是按照刚刚的步骤生成最终答案，如果验证失败则中断节点，人工介入生成正确的sql语句重新验证，如果成功则按照刚刚的步骤生成答案，还不成功则结束当前对话，提示：sql验证失败，无法回答。
